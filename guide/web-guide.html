⚙️ آليات عمل المنصة بالكامل

🔐 1. نظام المصادقة (Authentication)

يتم تسجيل المستخدمين عبر البريد الإلكتروني وكلمة المرور. تُخفظ البيانات في قاعدة بيانات آمنة مع تشفير باستخدام BCrypt. عند تسجيل الدخول، يُنشأ رمز جلسة (Token) يُخزن في الـ localStorage لاستمرار الجلسة.

📦 2. إدارة الخدمات (Services Management)

يمكن للبائع إضافة خدمة جديدة عبر نموذج يحتوي على عنوان، وصف، سعر، فئة، وصورة. تُضاف الخدمة فوراً للواجهة مع حفظها في قاعدة بيانات Supabase. كل خدمة تُعرض حسب تصنيفها مع ترتيب حسب التقييم أو السعر.

🛒 3. دورة الطلب (Order Flow)

عندما يطلب المشتري خدمة، تُخصم القيمة من محفظته وتُضاف 90% منها إلى محفظة البائع (10% عمولة المنصة). يتم تسجيل الطلب في قاعدة البيانات وإرسال إشعار للبائع عبر نظام الرسائل الداخلي.

💬 4. نظام الرسائل الفورية (Chat)

كل طلب يُنشئ محادثة تلقائية بين المشتري والبائع. يتم إرسال رسالة ترحيبية بعد الطلب. الرسائل تُخزن في قاعدة البيانات وتُعرض في واجهة دردشة مباشرة.

💰 5. المحفظة والمدفوعات (Wallet)

لكل مستخدم محفظة إلكترونية. عند إنشاء حساب جديد، يُمنح رصيد افتتاحي تجريبي (2000 جنيه). يمكن شحن المحفظة عبر وسائل دفع مصرية: فودافون كاش، InstaPay، أورانج كاش، اتصالات كاش، We Pay، وتحويل بنكي.

✅ 6. التوثيق والتحقق (Verification)

لتوثيق الحساب، يرفع المستخدم صورته الشخصية وصورة البطاقة. تُرفع الملفات إلى Amazon S3 ويُخزن رابط الملف في Supabase. عند اكتمال كلا الملفين، يُوثق الحساب تلقائياً.

⭐ 7. التقييمات والمراجعات (Reviews)

بعد إتمام الطلب، يمكن للمشتري تقييم الخدمة (1-5 نجوم) مع تعليق نصي. التقييمات تُخزن في سجل خاص وترتبط بمعرّف الخدمة لتعرض على صفحة الخدمة.

❤️ 8. المفضلة (Favorites)

يمكن للمستخدم إضافة خدمات إلى قائمة المفضلة بنقرة واحدة. المفضلة مرتبطة بحساب المستخدم وتُخزن في قاعدة البيانات لاسترجاعها في أي وقت.

🔑 9. صلاحيات الأدمن (Admin Panel)

لوحة الأدمن تتيح إدارة المستخدمين، مراجعة طلبات التوثيق، عرض الطلبات، إدارة النزاعات، ومتابعة المالية. يمكن للأدمن قبول أو رفض طلبات التوثيق وتعليق الحسابات المخالفة.

🗄️ 10. قاعدة البيانات والتخزين (Supabase + AWS)

جميع البيانات تُخزن في Supabase (PostgreSQL) كقاعدة بيانات أساسية. كل جدول يمثل كياناً: Users, Services, Orders, Wallets, Reviews, Messages, Favorites, Verifications. الملفات والصور تُخزن على Amazon S3 مع روابط مباشرة في قاعدة البيانات.

📋 آلية عمل Supabase + AWS

  1. Supabase Client — يتصل بقاعدة PostgreSQL عبر @supabase/supabase-js مع Row Level Security (RLS) لحماية البيانات.
  2. Auth (Supabase) — إدارة المصادقة عبر البريد الإلكتروني وكلمة المرور مع جلسات JWT آمنة.
  3. Storage (AWS S3) — رفع الصور والملفات إلى S3 Bucket عبر Presigned URLs لضمان الأمان.
  4. Edge Functions (Supabase) — دوال خادم بدون حالة لمعالجة الطلبات المعقدة كالمدفوعات والتوثيق.
  5. Serverless (AWS Lambda / Netlify) — دوال API لربط الواجهة بقاعدة البيانات ومعالجة المنطق التجاري.
  6. Realtime Subscriptions — اشتراكات مباشرة من Supabase لتحديث الدردشة والإشعارات بشكل فوري.
🧠 آلية الربط بين الواجهة والخادم (Frontend ↔ Backend)

تستخدم الواجهة مكتبة supabase-js للاتصال المباشر بقاعدة البيانات عبر RLS. للعمليات المعقدة، تُستخدم دوال Netlify Functions أو Supabase Edge Functions. كل طلب يمر عبر JWT Token للتحقق من هوية المستخدم. الصور والملفات تُرفع مباشرة إلى AWS S3 عبر Presigned URL من دون المرور بالخادم.

💾 استراتيجية تخزين البيانات

تُستخدم Supabase (PostgreSQL) كقاعدة بيانات أساسية مع علاقات بين الجداول (Foreign Keys) لضمان تكامل البيانات. للصور والملفات، يُستخدم AWS S3 للتخزين مع روابط مؤقتة (Presigned URLs). هذا يوفر أماناً عالياً، سرعة، وقابلية توسع لا نهائية.

🛒 الخدمات / Marketplace

📊 لوحة التحكم

إدارة الطلبات، الدفعات، البيانات الشخصية، ومتابعة المشاريع (بائع/مشتري).

⚙️ لوحة الأدمن

صلاحيات مركزية كاملة لإدارة المحتوى والمراجعة والإشراف على المنصة.

طريقة الاستخدام المقترحة للمستخدم الجديد

  1. ينشئ حساب جديد ويحدد نوعه (مشتري أو بائع).
  2. يكمل البيانات الأساسية داخل الملف الشخصي.
  3. يتصفح الخدمات أو البائعين ويختار الأنسب.
  4. ينفذ الطلب ويتابع التنفيذ من شاشة الطلبات.
  5. يضيف تقييم بعد الانتهاء لضمان الجودة.
💡 نصيحة تشغيل سريعة

ابدأ دائمًا باختبار تسجيل الدخول ثم تحميل صفحة البائعين ثم تنفيذ طلب تجريبي كامل للتأكد من دورة العمل End-to-End.
